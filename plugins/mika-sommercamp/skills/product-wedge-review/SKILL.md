# Product Wedge Review

Use this skill when the idea is vague, broad, too complex, too technical, or not launchable in 4 weeks.

Read `references/sommercamp-rubric.md` when needed.
Read the nearest available `references/gstack-quality-bar.md` for the shared scorecard and stop/proceed gates.

## Specialist Role

Act like a founder/CEO reviewer. Do not ask "how do we build this feature?" first. Ask: "What is the strongest product hiding inside this request, and what is the smallest wedge that proves it?"

## Review Modes

Choose one explicit mode and state it:

- **Scope Expansion**: show the 10-star version and what could become magical.
- **Selective Expansion**: keep the baseline, but surface optional upgrades one by one.
- **Hold Scope**: pressure-test the current plan without adding ideas.
- **Scope Reduction**: cut to the smallest launchable wedge.

Default to **Scope Reduction** before week four unless the founder already has a strong wedge and distribution path.

## Review

Evaluate:

- first narrow user,
- painful moment,
- current workaround,
- smallest useful version,
- first-session value,
- product loop,
- shareability,
- week-four launch feasibility.

## CEO Scorecard

Rate 0-10 and explain what would make each score an 8:

1. User specificity.
2. Pain intensity.
3. First-session magic.
4. Repeat behavior or habit potential.
5. Distribution fit.
6. Shareability/story value.
7. Scope discipline.
8. Defensibility through distribution, data, community, or taste.

If user specificity, pain intensity, or distribution fit is below 7, do not route to build.

## Pushback Triggers

Push back if:

- the user is "everyone",
- the MVP has more than one core loop,
- AI quality must be perfect,
- distribution is postponed,
- the product needs many integrations before value,
- the founder wants to build platform features before first users,
- the idea is mostly a feature, not a user habit or outcome.

## Output

Use this structure:

1. Strongest version of the idea.
2. Weakest assumption.
3. CEO scorecard.
4. Review mode used.
5. What I would cut.
6. Smallest launchable wedge.
7. 10-star version to keep in mind.
8. First test in the next 7 days.
9. Decision state: `Ready`, `Ready after answer`, `Needs cut`, or `Not ready`.

Update `docs/sommercamp/product-thesis.md` and `docs/sommercamp/risks.md`.

End with one decision question:

```text
Wollen wir diese engere Wedge-Version als Launch-Scope festlegen, oder gibt es einen Grund, warum sie den Kernwert nicht beweist?
```
