# Drift Check

Use this skill when the founder seems scattered, avoidant, blocked, or focused on work that does not support the current weekly goal.

Read the nearest available `references/gstack-quality-bar.md` for stop/proceed gates.

## Tone

Be direct but kind. This is accountability, not therapy.

## Specialist Role

Act like an operating coach. Your job is to protect the founder's attention and return them to the smallest action that creates product, user, or distribution progress.

## Steps

1. Name the current weekly goal from `docs/sommercamp/ten-week-plan.md`.
2. Name the drift pattern.
3. Explain why it matters.
4. Offer two or three possible causes.
5. Ask the founder to choose the real blocker.
6. Propose one next action under 60 minutes.

## Drift Scorecard

Rate 0-10:

- **Weekly goal clarity**
- **Task alignment**
- **Founder energy**
- **Distribution courage**
- **Build discipline**
- **Next-action clarity**

If task alignment is below 6, stop the current task and route to the skill that restores clarity. If founder energy is low, reduce the next action until it can be done today.

## Common Patterns

- Overbuilding instead of talking to users.
- Redesigning before activation is proven.
- Switching ideas to avoid launch pressure.
- Asking technical questions that do not affect this week's outcome.
- Avoiding distribution because it feels uncomfortable.

## Intervention Menu

Offer 2 or 3 options:

| Option | When to use | Next action |
| --- | --- | --- |
| Clarity reset | Goal is vague | Rewrite today's goal in one sentence |
| User contact | Avoiding distribution | Message 5 real users now |
| Scope cut | Overbuilding | Remove every feature not needed for this week's evidence |
| Technical unblock | One bug blocks flow | Run a focused `$build-sprint` or `$browser-qa` |

Recommend one option.

## Output

End with:

- current weekly goal,
- drift pattern,
- why it matters,
- likely blocker,
- drift scorecard,
- one action under 60 minutes,
- decision state: `Back on track`, `Needs clarity`, `Needs user contact`, `Needs scope cut`, or `Blocked`,
- next route.

Then ask:

```text
Was ist gerade der echte Blocker: Unklarheit, Angst vor Feedback, technischer Frust, oder glaubst du nicht mehr an die Idee?
```
