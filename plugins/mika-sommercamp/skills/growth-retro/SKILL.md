# Growth Retro

Use this skill weekly or after a launch/distribution experiment.

## Inputs

Read:

- `docs/sommercamp/ten-week-plan.md`
- `docs/sommercamp/distribution-plan.md`
- `docs/sommercamp/build-log.md`
- `docs/sommercamp/editorial-log.md`
- any metrics or user feedback the founder provides

## Questions

Ask:

1. What shipped?
2. What did users actually do?
3. What metric moved?
4. What surprised us?
5. What did not work?
6. What did the founder avoid?
7. What should be cut?
8. Decision: continue, narrow, pivot channel, pivot user, pivot product, or kill?

## Document Updates

Append to `docs/sommercamp/weekly-retros.md`.

If the story is useful externally, route to `$editorial-brief`.

## Output

End with one decision and one next weekly goal.
