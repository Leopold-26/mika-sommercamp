# Sommercamp Rubric

Use this rubric to evaluate product ideas, weekly plans, and launch readiness.

## Strong Idea Signals

- Narrow first user.
- Painful or frequent use case.
- Simple first product loop.
- Clear first-session value.
- Credible first-user channel.
- Product creates a shareable artifact or story.
- Founder has a personal advantage in the market, user group, or channel.
- Week-four launch scope is realistic.

## Weak Idea Signals

- "Everyone" is the user.
- Requires many integrations before value exists.
- Needs perfect AI quality to work.
- Looks more like B2B workflow software than a consumer app.
- No clear reason users return.
- Distribution is postponed until after building.
- Founder cannot explain how first 100 users arrive.
- Scope depends on a large team.

## Review Questions

1. Can this be useful with one core feature?
2. Can a first user get value in under 2 minutes?
3. What would be embarrassing or risky at public launch?
4. What must be measured from day one?
5. What can be cut without weakening the core test?
6. What assumption could kill the idea fastest?

## Default Recommendation Logic

- If target user is broad: narrow the segment.
- If product is broad: define the wedge.
- If distribution is weak: run distribution review before build sprint.
- If scope is large: cut to week-four launch.
- If founder is blocked by tech: translate into a Codex build sprint.
- If founder is blocked by confidence or focus: run drift check.
